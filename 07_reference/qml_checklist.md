# Чек-лист по QML

## Перед тем как писать новый экран

- понятно ли, какие данные нужны экрану
- какие части можно вынести в компоненты
- где будет жить состояние
- что должно обновляться автоматически через binding

## Перед тем как переносить что-то в C++

- есть ли здесь бизнес-логика
- связано ли это с данными, файлами, сетью или устройством
- нужна ли эта логика в нескольких местах
- становится ли QML слишком сложным для чтения
